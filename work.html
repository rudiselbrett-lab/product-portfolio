SMB Digital Presence Agent

Active

Python · Claude API · Playwright · Streamlit · Pydantic v2 · HTTPX

The Problem

Most small service businesses in Charlotte — plumbers, roofers, HVAC, electricians — are invisible in local search and AI-powered recommendations. Roughly 35–40% of local service businesses have no website at all; of those that do, 60–70% haven't been updated in over three years. AI search tools increasingly surface businesses with structured web presence over those without. This gap gets larger every year.

The Core Insight

The real split is binary: has a website (conversation is optimization — SEO, speed, conversion rate) vs. no website (conversation is existence — invisible to AI search and under-indexed in Google Maps). Building a system that detects this split and routes to different outreach angles is the central architectural decision. Same pipeline, two messages, meaningfully higher reply rate.

What I Built

An end-to-end multi-agent pipeline that discovers Charlotte SMBs via Google Maps scraping (no API cost), routes each into Track A (has website → optimization analysis) or Track B (no website → digital gap analysis), scores each as an opportunity 0–100, discovers owner contact info, and drafts track-specific cold emails — all queued for human review before anything sends.

Why I Built It

Practice for agent engineering. The real design questions: how do you structure data flow across agents without it becoming a mess? Where does Claude add value vs. where is a heuristic good enough? How do you build a pipeline that degrades gracefully when API keys are missing? What does a human-in-the-loop system actually look like? The SMB problem is real — but the primary output is a working agent system I understand end-to-end.
