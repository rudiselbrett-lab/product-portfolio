SMB Digital Presence Agent

Active

Python · Claude API · Playwright · Streamlit · Pydantic v2 · HTTPX

The Problem

Most small service businesses in Charlotte — plumbers, roofers, HVAC companies, electricians — have a broken relationship with the internet. They're either completely absent or running a site built in 2014 that hasn't been touched since. The result: invisible in local search, invisible in AI-powered recommendations, losing inbound leads to competitors who spent $500 on a basic site.

Roughly 35–40% of local service businesses have no website at all. Of those that do, 60–70% haven't been updated in over three years. AI search tools increasingly surface businesses with structured web presence over those without — and that gap widens every year.

The Core Insight

The real split is binary: has a website (conversation is optimization — SEO, speed, conversion rate, mobile UX) vs. no website (conversation is existence — you are invisible to AI search and under-indexed in Google Maps). Building a system that detects this split and routes to different outreach angles is the central architectural decision. Same pipeline, two messages, meaningfully higher reply rate.

What I Built

An end-to-end multi-agent pipeline that discovers Charlotte SMBs via Google Maps scraping (no API cost), routes each into Track A or Track B based on whether a live website is detected, analyzes the appropriate gaps, scores each as an opportunity 0–100, discovers owner contact information, drafts track-specific cold emails, and queues everything for human review before any outreach goes out.

Where It Stands

Pipeline runs end-to-end on real Charlotte data. The Google Maps scraper pulls businesses without an API key, the website router correctly identifies live vs. missing sites, the analysis agents produce scored outputs, and the email drafts are branched by track. A Streamlit dashboard lets me run a batch, review leads, and mark emails as sent.

What's next: validate scraper output quality against real reply rates, tune scoring thresholds, add Gmail API sending, and track conversion by track (A vs. B) to validate the two-angle hypothesis.

Why I Built It

Practice for agent engineering. The real design questions: how do you structure data flow across agents without it becoming a mess? Where does Claude add value vs. where is a heuristic good enough? How do you build a pipeline that degrades gracefully when API keys are missing? What does a human-in-the-loop system actually look like in practice? The SMB problem is real — but the primary output is a working agent system I understand end-to-end, not a business I'm ready to scale.

Captain's Log

Running build log — what's shipping, what's breaking, what's next.

June 29, 2026 Session 1
What We Built

Started from a blank repo. Built a two-track SMB discovery system in Python. The core insight driving the whole thing: businesses either have a website or they don't — and those are two completely different sales conversations. The pipeline routes every business into Track A (has a website, optimization play) or Track B (no website, greenfield capture). Each track gets its own analysis, its own scoring, and its own email angle.

The pipeline runs end to end — scrapes Google Maps for local businesses, checks for a website, analyzes what's there or what's missing, scores the opportunity, and drafts a cold email. Everything lands in a review queue before anything gets sent. Built a Streamlit dashboard on top so you can run it, see results, and review emails without touching the command line.

What's Working

The structure is solid. Agent architecture, data models, two-track routing, scoring engine, review queue — all of that works. The Streamlit UI runs and the pipeline flows end to end.

The Google Maps scraper pulls real Charlotte businesses using Playwright with no API key required. Got that working today.

What We're Still Debugging

The scraper is the live piece we're actively tuning. Two problems hit today:

First — Google Maps doesn't always list a website even when a business has one. Built a Google search fallback: if Maps comes up empty, search for the business name and look for their real site in organic results.

Second — that fallback was too loose. It was returning false positives, crediting businesses with websites that weren't actually theirs. Fixed by requiring the domain to contain words from the business name before accepting it.

What's Next

Validate the scraper is returning clean data consistently. Once that's solid, add the Anthropic key to get real AI-scored analysis and proper personalized emails instead of template copy. After that — test actual reply rates on Track A vs Track B to see if the two-angle hypothesis holds.
